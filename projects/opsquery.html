OpsQuery

AI-powered natural-language-to-SQL assistant for Canterbury Dairy Co.

Live demo — OpsQuery in action

An AI-powered natural language to SQL assistant built for Canterbury Dairy Co, a fictional dairy processing company in Canterbury, New Zealand. Non-technical staff — operations managers, quality leads, and logistics coordinators — can ask business questions in plain English and receive instant answers from a live operational database. No SQL knowledge required.

How It Works

OpsQuery was built to solve a real bottleneck in data-driven operations: the dependency on developers or manual reporting to answer routine questions. In dairy processing, decisions about farm performance, milk collection quality, and route efficiency happen daily — and slow data access has a direct cost.

The tool takes a plain English question, passes it through an input security validator that blocks injection attempts, sends it to the Claude AI API which generates a PostgreSQL query, validates that the query is SELECT-only, then executes it against a read-only database user and returns the result as an interactive table.

"Which farms failed quality tests this week?"

This separation of concerns — security layer, AI layer, database layer — mirrors how production data systems are designed in industry.

Security

Security was designed into the system from the start, not added later. Input validation blocks 15+ SQL keywords before the question reaches the AI. Generated queries are checked to confirm they are SELECT-only. A dedicated read-only PostgreSQL user ensures the AI cannot modify or delete data. Credentials are stored in environment variables and excluded from version control.
